What is pair programming?

Pair programming is a method of collaborative programming used by Dev Bootcamp, employers, and other programs to provide better and more reliable results. It employs each person taking on a role of either Driver or Navigator. Each has different responsibilities but both are active in the process.

Why is pair programming important to Dev Bootcamp?

Pair programming is important to Dev Bootcamp because not only is it a method that will be used in our jobs after the course is over, it also instills many of the philosophies that Dev Bootcamp believes in. Pair programming builds empathy between the individuals but it also prepares them for continuous pairing with other, different people with different backgrounds.

Comparison between Driver and Navigator

  • Driver
  • Manipulates keyboard
  • Actively coding
  • Sharing thought process of code
  • Navigator
  • Actively reviewing code
  • Researching code
  • Reviewing reference materials
  • Generally guiding direction of code
